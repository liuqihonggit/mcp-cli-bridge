# NET10.0工程管理规范

## 1. 全局化工程模板

### 1.1 Directory.Build.props

- 集中式属性管理，统一编译参数、版本号、代码分析规则
- **强制NativeAOT编译**，这是不可变原则
- 项目打包到npm，非AOT会导致安装崩溃

### 1.2 GlobalUsings.cs

- 全局命名空间统一导入
- `.cs`代码文件内**禁止**书写using引用，一律搬运到GlobalUsings.cs
- 命名空间冲突用简短别名在GlobalUsings.cs澄清

### 1.3 AOT兼容性要求

- 所有代码必须围绕NativeAOT编译构造
- JSON处理必须用类表示，禁止直接书写和解析JSON字符串
- 动态行为改为：直接写死(优先) 或 源码生成器(仅当前项目内使用)

## 2. 项目文件结构

### 迁移前读取

你要渐进式移动,每移动一次都编译和备份,实现阶段性胜利.
不然可能导致你一直回滚,然后任务没有完成.
子智能体可能会重复你的迁移错误,因此你只能用子智能体进行修改命名空间,否则你不知道子智能体出错内容是什么.

#### PowerShell Set-Content 导致文件损坏

- **错误编号**: CS1022
- **问题场景**: 使用 PowerShell 的 `Set-Content` 命令批量替换 C# 文件内容时，导致文件损坏，出现 CS1022 编译错误
- **原因**: PowerShell 的 `Set-Content` 在处理多行文本时可能出现编码或格式问题，导致文件内容被破坏
- **解决方案**: 
  1. 使用分阶段迁移策略：先移动文件保持原有命名空间，确保编译通过后再调整命名空间
  2. 使用 SearchReplace 工具代替 PowerShell Set-Content 来修改文件内容
  3. 对于批量文件操作，优先使用 C# 代码专家 agent 处理
- **验证结果**: 文件恢复后重新采用分阶段迁移策略成功
- **标记**: 【避坑指南】

## 3. 服务架构规范

### 3.1 主机项目(Host)

- 引用基建契约和基建实现
- 引用其它子服务契约和子服务实现
- 通过IoC核心冷启动
- 中间件实现AOP
- 全局拦截异常，带日志栈帧标记
- 每个外部请求调用都要try-catch，并记录日志
- 性能监控埋点

### 3.2 基建项目(Common)

- 基建项目包含多个子模块：Caching、CliProtocol、Configuration、Constants、IoC、Json、Logging、Middleware、Plugins、Reflection、Results、Security、Tools
- 每个子模块作为Common项目的子目录，而不是独立项目
- **Contracts项目只能有接口/抽象，不能有具体实现**
- 纯DTO（只有属性，无业务逻辑）符合Contracts定位
- Common项目输出DLL，被McpHost和Plugins引用
- 统一封装全局基建工具类，所有文件读写强制收口
- 内置跨进程Mutex全局锁、进程内读写锁、CancellationToken超时控制
- 集成全局性能监控埋点，统计读写耗时与异常
- **Json模块**：提供AOT兼容的JSON序列化服务

### 3.3 独立NuGet包(McpProtocol)

- MCP协议解析作为独立的NuGet包发布
- 位于`lib/McpProtocol/`目录下
- 分为契约(McpProtocol.Contracts)和实现(McpProtocol)
- 可以被其他项目引用，不依赖主项目
- 输出DLL，支持NativeAOT编译

### 3.4 主机服务(McpHost)

- 主机服务项目，输出EXE
- 引用Common、Common.Contracts、McpProtocol（NuGet包）
- 通过IoC核心冷启动
- 中间件实现AOP
- 全局拦截异常，带日志栈帧标记
- 每个外部请求调用都要try-catch，并记录日志
- 性能监控埋点
- 管理Plugins目录下的CLI服务
- 通过ProcessPool管理CLI进程池

### 3.5 CLI服务(Plugins)

- 每个CLI服务是独立的EXE项目，输出EXE
- CLI服务通过CLI协议与McpHost通信
- CLI服务内部通过说明文档调用，降低上下文成本
- CLI服务工具不直接暴露，通过`tool_execute`间接调用
- CLI服务目录结构：
  - `Commands/` - 命令处理器
  - `Services/` - 业务服务
  - `Validation/` - 参数验证（可选）
  - `CLI说明.md` - CLI使用说明
- CLI服务只能引用Common、Common.Contracts
- CLI服务不能直接引用McpHost

## 4. 插件系统规范（CLI架构）

### 4.1 CLI插件通信协议

```csharp
// CLI插件通过标准输入输出通信
// 请求格式：JSON-RPC 2.0
{
    "jsonrpc": "2.0",
    "method": "tool_name",
    "params": { ... },
    "id": 1
}

// 响应格式
{
    "jsonrpc": "2.0",
    "result": { ... },
    "id": 1
}
```

### 4.2 CLI插件生命周期

- McpHost启动时扫描Plugins目录下的CLI.exe
- 通过ProcessPool管理CLI进程池
- 每个CLI进程处理多个请求后回收
- 插件加载失败时记录详细日志，不影响主程序运行

### 4.3 CLI插件安全

- 插件运行在独立进程隔离
- 通过ProcessPool限制资源使用
- 插件超时自动终止
- 插件崩溃不影响McpHost运行

## 7. 消息总线(MQ)预留设计

### 7.1 设计原则

- MQ仅用于跨进程通讯，内部通过契约直接调用
- 预留接口设计必须满足未来扩展需求

### 7.2 消息接口抽象

```csharp
public interface IMessageBus
{
    Task PublishAsync<T>(T message, CancellationToken ct) where T : IMessage;
    Task<TResponse> RequestAsync<TRequest, TResponse>(TRequest request, CancellationToken ct) 
        where TRequest : IRequest<TResponse>;

// 消息基类
```

## 8. 死锁防护

### 8.1 分片锁规范

- **必须排序后使用**，避免循环等待
- **超时退出**，所有锁必须提供超时机制
- **记录日志**，锁获取失败必须记录原因

### 8.2 锁使用示例

```csharp
// 错误：无序获取
lock (resourceA) { lock (resourceB) { } }
lock (resourceB) { lock (resourceA) { } } // 死锁！

// 正确：排序后获取
var ordered = new[] { resourceA, resourceB }.OrderBy(r => r.Id);
foreach (var r in ordered) { /* 获取锁 */ }
```

## 10. 编码规范

### 10.1 反射与硬编码

**❌ 禁止硬编码数组：**

```csharp
private static readonly string[] Commands = new[] { "add", "commit", "push" };
```

**✅ 使用特性+源码生成器（AOT兼容方式）：**

```csharp
[Command("add")]
public class AddCommand { }

// 必须使用编译时源码生成器，禁止运行时反射
// NativeAOT不支持反射emit和动态代码生成
```

### 10.2 参数封装

**❌ 避免多参数：**

```csharp
public void Test(string cmd, string args, string options) { }
```

**✅ 封装为类：**

```csharp
public void Test(Command cmd) { }
// 解析不发生在总线
```

### 10.3 字典替代硬编码

**❌ 禁止if-else链：**

```csharp
if (type == "A") return 1;
if (type == "B") return 2;
```

**✅ 使用字典：**

```csharp
private static readonly Dictionary<string, int> TypeMap = new()
{
    ["A"] = 1,
    ["B"] = 2
};
```

**✅ AOT优化：使用FrozenDictionary：**

```csharp
using System.Collections.Frozen;

private static readonly FrozenDictionary<string, int> TypeMap = 
    new Dictionary<string, int>
    {
        ["A"] = 1,
        ["B"] = 2
    }.ToFrozenDictionary();
```

### 10.4 GC优化

- 字符串操作优先使用`Span<T>`
- 采用`HashSet`/`Dictionary`替代`List`/`Array`
- 忽略大小写比较时使用`StringComparer.OrdinalIgnoreCase`，避免转大小写
- AOT优化：使用`FrozenDictionary`提升性能

### 10.5 LINQ链式编程

**❌ 避免命令式循环：**

```csharp
var results = new List<int>();
foreach (var item in source)
{
    if (item > 0)
        results.Add(item * 2);
}
```

**✅ 使用LINQ链式：**

```csharp
var results = source
    .Where(x => x > 0)
    .Select(x => x * 2)
    .ToList();
```

### 10.6 类型安全引用

**❌ 禁止字符串硬编码：**

```csharp
var typeName = "MyClass";
var propertyName = "Name";
```

**✅ 使用类型安全操作符：**

```csharp
var typeName = nameof(MyClass);
var propertyName = nameof(MyClass.Name);
```

## 14. 项目特定要求

### 14.1 McpHost项目定位

- 仅支持MCP接口
- 通过少量MCP查找CLI.exe工具
- 每个CLI.exe内部通过说明文档调用，降低上下文成本
- CLI内部工具不直接暴露，通过`tool_execute`间接调用

### 14.2 插件发布

- 通过`build.ps1`脚本发布插件到`publish/`文件夹
- 包括npm包相关内容和readme.md
- 不需要配置文件，查找同目录下的CLI.exe
- 发布时不能手动复制文件，只能修改生成逻辑

### 14.3 CLI服务目录隔离

- `src/Plugins/` 文件夹下是CLI服务
- 每个CLI服务是独立的EXE项目
- CLI服务不能直接引用McpHost项目
- CLI服务只能引用Common、Common.Contracts
- McpHost通过CLI协议与CLI服务通信，不直接调用CLI服务代码

## 15. 技术债预防检查清单

| 检查项          | 要求                                 |
| ------------ | ---------------------------------- |
| AOT编译        | 所有项目必须支持NativeAOT编译                |
| JSON处理       | 禁止直接解析JSON字符串，必须用类表示               |
| GlobalUsings | .cs文件内禁止using语句，统一在GlobalUsings.cs |
| Contracts项目  | 只能有接口/抽象，不能有具体实现                   |
| CLI服务隔离      | CLI服务不能直接引用McpHost，只能通过CLI协议通信     |
| 参数封装         | 避免多参数方法，封装为类                       |
| 禁止硬编码        | 使用字典/特性替代if-else链和硬编码数组            |
| 超时控制         | 所有异步操作必须传入CancellationToken        |
| 异常处理         | 每个外部请求都要try-catch并记录日志             |
| 性能监控         | 关键路径必须埋点监控性能                       |
| 基建项目         | 所有共享功能必须放在Common项目中                |
| 服务项目         | CLI服务输出EXE，不能输出DLL                 |
| 独立NuGet包     | MCP协议解析必须作为独立NuGet包发布              |
