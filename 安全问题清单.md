## 2. CLI 插件自动加载风险

### 问题描述
`PackageManager.DiscoverAvailablePlugins()` 会自动扫描并加载以下目录的所有 `.exe` 文件：
- `{主程序目录}/plugins/`
- `{主程序目录}/`

### 代码位置
- `src/McpHost/Services/PackageManager.cs:76-128`

### 潜在攻击场景

| 攻击场景 | 前提条件 | 风险等级 |
|---------|---------|---------|
| 恶意 .exe 放入 plugins 目录 | 需要文件写入权限 | 中 |
| 供应链攻击（npm 包被篡改）| 使用 `package_install` 下载 | 中 |
| LLM 被诱导调用恶意工具 | 恶意插件已加载 | 高 |

### 风险评估
- 如果攻击者已有文件写入权限，可直接执行恶意程序，无需通过 McpHost
- 当前安全边界 = 文件系统权限

### 轻量级保护方案

#### 方案 A: 插件白名单配置
```json
// mchost.config.json
{
  "allowedPlugins": ["MemoryCli", "FileReaderCli"]
}
```

#### 方案 B: 启动确认
```
[WARN] Discovered unknown plugin: UnknownCli.exe
       Allow loading? (y/N)
```

#### 方案 C: 签名验证
- 插件需带有有效数字签名
- 验证通过才允许加载

## 3. npm 包下载风险

### 问题描述
`package_install` 工具通过 npm pack 下载包并解压到本地。

### 代码位置
- `src/McpHost/Services/PackageManager.cs:130-176`

### 潜在风险
- npm 包可能被篡改（供应链攻击）
- 解压的文件可能包含恶意可执行文件

### 缓解措施建议
- 验证包签名
- 限制可安装的包源
- 沙箱隔离执行

