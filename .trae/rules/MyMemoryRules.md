## 避坑指南

### PowerShell Set-Content 导致文件损坏

- **错误编号**: CS1022
- **问题场景**: 使用 PowerShell 的 `Set-Content` 命令批量替换 C# 文件内容时，导致文件损坏，出现 CS1022 编译错误
- **原因**: PowerShell 的 `Set-Content` 在处理多行文本时可能出现编码或格式问题，导致文件内容被破坏
- **解决方案**: 
  1. 使用分阶段迁移策略：先移动文件保持原有命名空间，确保编译通过后再调整命名空间
  2. 使用 SearchReplace 工具代替 PowerShell Set-Content 来修改文件内容
  3. 对于批量文件操作，优先使用 C# 代码专家 agent 处理
- **验证结果**: 文件恢复后重新采用分阶段迁移策略成功
- **标记**: 【避坑指南】
